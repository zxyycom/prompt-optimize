# Agent Instructions

## 项目边界

1. 本仓库用于维护 `prompt-optimize` Codex skill。skill 本体放在 `skill/prompt-optimize/`。
2. 根目录、`docs/`、`scripts/` 和 `.github/` 只承接项目级说明、维护约定、决策记录、工具链说明、自动化脚本、CI 和仓库元数据。
3. 项目文档只写本仓库内可维护的信息；仓库外路径、临时来源和一次性调查上下文不进入长期文档。

## 维护原则

1. `AGENTS.md` 只保留长期稳定的项目级约定。具体文件职责写在 README 或对应文件中, skill 行为细节写在 skill 本体中。
2. 本项目规模小, 修改前快速扫当前项目文件和相关上下文即可, 不维护固定读取清单。
3. 保持改动范围与用户请求对齐, 不为了整理文档而重写无关内容。
4. 新增长期规则前先判断 owner；只有当前文件适合承接项目级约定时, 才修改 `AGENTS.md`。
5. 脚本、安装、校验、打包和 CI 的细节由 `docs/tooling.md` 承接; `AGENTS.md` 只保留 owner 引用。
6. 正文主要使用中文；除用户要求或目标文件已有语言要求外, 新增内容保持中文。

## 决策记录

1. 决策记录只记录后续维护需要回放原因的判断, 不作为变更日志。
2. skill 行为、触发条件、读取策略、引用 owner、规则边界或验收标准发生变化时, 按 `docs/decisions/maintenance.md` 判断是否达到记录门槛。
3. 项目级决策只在改变长期维护契约、目录边界、自动化交付方式或跨文件 owner 时记录。
4. 普通文字修正、格式调整、链接修复和按既有规则执行的一次性细节不记录。
